# Release 0.1.17 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Smart 0.1.17 with complete Chinese workflow artifact instructions and synchronized package, lockfile, manifest, CLI, and Git tag versions.

**Architecture:** Workflow language behavior remains prompt-driven in the four build/verify skill files and is protected by contract tests. Package and CLI versions continue to use `package.json` as the runtime source, while the distributable asset manifest is synchronized explicitly before tagging.

**Tech Stack:** Markdown skills, TypeScript, Vitest, npm, GitHub Actions, npm registry

---

### Task 1: Protect Chinese Artifact Language Rules

**Files:**
- Create: `test/artifact-language.test.ts`
- Modify: `assets/skills/smart-build/SKILL.md`
- Modify: `assets/skills-zh/smart-build/SKILL.md`
- Modify: `assets/skills/smart-verify/SKILL.md`
- Modify: `assets/skills-zh/smart-verify/SKILL.md`

- [ ] **Step 1: Write the failing contract test**

Create a table-driven Vitest test that reads both build skills and both verify skills. Assert every file contains `.smart/config.yaml`, `smart_language`, `zh`, `en`, user-request fallback, existing-artifact preservation, and unchanged paths/file names. Assert build files name `docs/superpowers/plans/` and `tasks.md`; assert verify files name `verification-report.md` and `tasks.md`.

- [ ] **Step 2: Verify the contract fails for the missing rules**

Run: `npx vitest run test/artifact-language.test.ts`

Expected: FAIL because the build and verify skill files do not yet contain the complete resolution contract.

- [ ] **Step 3: Add minimal language sections to all four skills**

Add equivalent English and Chinese instructions stating:

```text
Read .smart/config.yaml before creating or updating artifacts.
smart_language: zh => Chinese; smart_language: en => English.
If absent, use the triggering request language.
When resuming, preserve the dominant existing artifact language unless explicitly switched.
Localize prose only; keep paths, file names, metadata keys, commands, identifiers, and machine-readable values unchanged.
```

Apply the resolved language to plans and `tasks.md` during build, and to `verification-report.md` plus accepted-deviation updates to `tasks.md` during verify.

- [ ] **Step 4: Verify the focused contract passes**

Run: `npx vitest run test/artifact-language.test.ts`

Expected: PASS.
